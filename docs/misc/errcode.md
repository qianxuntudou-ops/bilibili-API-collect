## 风控类错误码详解

以下错误码与风控相关，触发时可能需要特殊处理：

### -352 风控校验失败

**触发原因：**
- User-Agent 为空或不合法
- Wbi 签名计算错误或过期
- 请求频率过高
- 缺少必要的 Cookie 字段
- Referer 异常
- 请求模式被识别为机器人

**解决方法：**
1. 检查并设置正确的 User-Agent
2. 重新计算 Wbi 签名
3. 补充必要的 Cookie（`bili_ticket`、`b_nut`、`buvid3`、`buvid4` 等）
4. 降低请求频率
5. 若以上方法无效，进行 [v_voucher 验证](../sign/v_voucher.md)

### -412 请求被拦截

**触发原因：**
- 客户端 IP 被服务端风控
- IP 请求频率过高
- IP 被标记为异常

**解决方法：**
1. 更换 IP 地址
2. 使用代理池
3. 降低请求频率
4. 等待一段时间后再试

### -799 请求过于频繁

**触发原因：**
- 请求频率超过接口限制
- 短时间内大量请求同一接口

**解决方法：**
1. 降低请求频率
2. 添加请求间隔（建议 1-3 秒）
3. 使用队列控制请求速率

## 风控参数说明

当触发风控时，接口可能返回以下参数：

| 参数名 | 位置 | 说明 |
|--------|------|------|
| `v_voucher` | data 对象或响应头 `x-bili-gaia-vvoucher` | 风控凭证，用于申请验证 |
| `gaia_res_type` | data 对象 | 风控结果类型 |
| `gaia_data` | data 对象 | 风控相关数据 |

**使用方式：**
- 获取 `v_voucher` 后，调用 `/x/gaia-vgate/v1/register` 申请验证
- 完成验证后，获取 `grisk_id` 作为 `gaia_vtoken` 和 `x-bili-gaia-vtoken`
- 在后续请求中添加这些参数以解除风控
